1. Create the custom Q app

This page walks through creating a custom Q application (skeleton) that will integrate with Jira.

1. Plan the integration

  • Decide which Jira operations you need.
  • Choose authentication method: OAuth1, OAuth2, or API token.

2. Create the new app in Q

  • Sign in to your Q developer console.
  • Create a new app/project and set redirect URI.

3. App skeleton

git init custom-q-jira-integration
package.json, src/

4. Secrets and storage

Never store private keys or client secrets in plaintext. Use secret managers.

5. Add Jira client code

  • createIssue(payload)
  • updateIssue(issueId, payload)
  • deleteIssue(issueId)
  • getIssue(issueId)

6. Local dev notes

JIRA_BASE_URL=https://your-domain.atlassian.net
JIRA_AUTH_METHOD=oauth2
JIRA_CLIENT_ID=
JIRA_CLIENT_SECRET=
CALLBACK_URL=http://localhost:3000/auth/callback
