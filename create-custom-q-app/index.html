    1. Create the custom Q app

    This page walks through creating a custom Q application (skeleton) that will integrate with Jira.

    1. Plan the integration
    2. Decide which Jira operations you need: create issues, update issues, delete issues, query issues, comments, transitions, webhooks, etc.

    3. Choose authentication method: OAuth 1.0a (RSA) for Server/DC, OAuth 2.0 (3LO) for Jira Cloud, or an API token (Cloud) for non-OAuth flows.

    4. Create the new app in Q (high-level steps)

    5. Sign in to your Q developer console.
    6. Create a new app/project and give it a name like custom-q-jira-integration.

    7. Set a redirect URI (if using OAuth 2.0) to your app's callback endpoint, e.g. https://<your-app>/auth/callback.

    8. App skeleton (example)

    9. Create a repository or local project:
    10. Initialize project: git init custom-q-jira-integration
    11. Create basic files: package.json, src/ (or the language equivalent)
    12. Add configuration placeholders for Jira:
    13. JIRA_BASE_URL
    14. JIRA_AUTH_METHOD (e.g., oauth1, oauth2, api_token)
    15. JIRA_CONSUMER_KEY (OAuth 1.0a)
    16. JIRA_PRIVATE_KEY (PEM) â€” store in a secure secret store
    17. JIRA_CLIENT_ID / JIRA_CLIENT_SECRET (OAuth 2.0)

    18. CALLBACK_URL (OAuth redirect URI)

    19. Secrets and storage

    20. Never store private keys or client secrets in plaintext in repo.

    21. Use Q platform secrets or your cloud provider secret manager for private keys and client secrets.

    22. Add Jira client code

    23. Implement a thin wrapper around Jira REST API (or use an existing SDK).
    24. Example endpoints to implement:
    25. createIssue(payload)
    26. updateIssue(issueId, payload)
    27. deleteIssue(issueId)

    28. getIssue(issueId)

    29. Local dev notes

    30. Use environment file .env for local testing (but do not commit).
    31. Example .env:
    32. JIRA_BASE_URL=https://your-domain.atlassian.net
    33. JIRA_AUTH_METHOD=oauth2
    34. JIRA_CLIENT_ID=
    35. JIRA_CLIENT_SECRET=
    36. CALLBACK_URL=http://localhost:3000/auth/callback
